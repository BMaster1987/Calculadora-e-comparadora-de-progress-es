# Interactive Progressions Laboratory

## Project Overview
This project is an interactive offline educational web application focused on mathematical progressions for high school students.

It was developed as a teaching and learning resource to support the study of:
- Arithmetic Progression (A.P.)
- Geometric Progression (G.P.)
- Harmonic Progression (H.P.)

The application was designed to be visually engaging, didactic, and practical for classroom use and individual study.

## Educational Purpose
This project was created for educational purposes, especially for high school students who are learning sequence patterns, formulas, graphs, and comparisons between different types of progressions.

Its main goals are:
- to make abstract concepts easier to visualize
- to provide immediate numerical and graphical feedback
- to support exploration, comparison, and pattern recognition
- to serve as an interactive learning tool for mathematics classes

## What Was Implemented
### 1. Professional interface redesign
- Cleaner visual hierarchy
- More consistent layout
- Standardized buttons and action areas
- Improved readability and user navigation
- More polished educational app appearance

### 2. Separate laboratories for each progression
- Arithmetic Progression laboratory
- Geometric Progression laboratory
- Harmonic Progression laboratory

Each laboratory includes:
- input parameters
- formula display
- generated terms table
- graph visualization
- quick examples
- table search
- export options

### 3. Comparison module
- Added a selector to compare:
  - only two progressions
  - or all three progressions at the same time
- Improved comparative reading of the results
- Better support for identifying differences between progression behaviors

### 4. Sequence analyzer improvements
- Fixed parsing issues for Brazilian decimal notation
- Inputs now support decimal commas
- Better handling of manually typed sequences
- More reliable behavior for student input

### 5. Graph improvements
- Better visual clarity
- Better label positioning
- Adjustments to avoid overlap between axis labels and numeric values
- Improved readability in all progression graphs
- More didactic presentation for interpretation

### 6. Brazilian decimal format support
- Numeric input follows Brazilian decimal conventions
- Decimal comma support was prioritized for usability in Brazil
- Input behavior was adapted for local educational context

### 7. Harmonic progression support
- Harmonic progression was added with features equivalent to the other progression modules
- Includes formulas, graph, generated table, examples, and exports
- Added educational interpretation through the associated arithmetic progression denominator idea

### 8. Quiz preservation
- The quiz section was intentionally kept focused only on:
  - Arithmetic Progression
  - Geometric Progression
- No major structural changes were made there, as requested

## Technical Notes
- The project was built as an offline HTML/CSS/JavaScript application
- A more modular version was also organized for better maintenance
- The interface and logic were refined across multiple iterations

# Laboratório Interativo de Progressões

## Visão Geral do Projeto

Este projeto é uma aplicação web educacional interativa offline, focada no estudo de progressões matemáticas para alunos do ensino médio.

Foi desenvolvido como um recurso de ensino e aprendizagem para apoiar o estudo de:

* Progressão Aritmética (P.A.)
* Progressão Geométrica (P.G.)
* Progressão Harmônica (P.H.)

A aplicação foi projetada para ser visualmente atrativa, didática e prática para uso em sala de aula e estudo individual.

## Objetivo Educacional

Este projeto foi criado com fins educacionais, especialmente para estudantes do ensino médio que estão aprendendo padrões de sequências, fórmulas, gráficos e comparações entre diferentes tipos de progressões.

Seus principais objetivos são:

* tornar conceitos abstratos mais fáceis de visualizar
* fornecer feedback numérico e gráfico imediato
* apoiar exploração, comparação e reconhecimento de padrões
* servir como uma ferramenta interativa para aulas de matemática

## O que foi implementado

### 1. Redesign de interface profissional

* Hierarquia visual mais limpa
* Layout mais consistente
* Padronização de botões e áreas de ação
* Melhor legibilidade e navegação
* Aparência mais refinada de aplicativo educacional

### 2. Laboratórios separados para cada progressão

* Laboratório de Progressão Aritmética
* Laboratório de Progressão Geométrica
* Laboratório de Progressão Harmônica

Cada laboratório inclui:

* parâmetros de entrada
* exibição de fórmulas
* tabela de termos gerados
* visualização gráfica
* exemplos rápidos
* busca na tabela
* opções de exportação

### 3. Módulo de comparação

* Adicionado seletor para comparar:

  * apenas duas progressões
  * ou as três progressões simultaneamente
* Melhor leitura comparativa dos resultados
* Melhor suporte para identificar diferenças entre os comportamentos das progressões

### 4. Melhorias no analisador de sequências

* Correção de problemas de leitura de valores no formato brasileiro
* Suporte a vírgula decimal
* Melhor tratamento de sequências digitadas manualmente
* Maior confiabilidade para uso pelos alunos

### 5. Melhorias nos gráficos

* Maior clareza visual
* Melhor posicionamento de rótulos
* Ajustes para evitar sobreposição entre valores e eixos
* Melhor legibilidade em todos os gráficos
* Apresentação mais didática para interpretação

### 6. Suporte ao formato decimal brasileiro

* Entrada numérica adaptada ao padrão brasileiro
* Prioridade para uso de vírgula decimal
* Comportamento ajustado para o contexto educacional no Brasil

### 7. Implementação da Progressão Harmônica

* Inclusão completa da Progressão Harmônica com funcionalidades equivalentes às outras
* Fórmulas, gráficos, tabela, exemplos e exportações
* Interpretação didática baseada na progressão aritmética dos denominadores

### 8. Preservação do quiz

* A seção de quiz foi mantida focada apenas em:

  * Progressão Aritmética
  * Progressão Geométrica
* Nenhuma alteração estrutural significativa foi feita, conforme solicitado

## Notas Técnicas

* O projeto foi desenvolvido como uma aplicação offline em HTML, CSS e JavaScript
* Uma versão mais modular também foi organizada para facilitar manutenção
* Interface e lógica foram refinadas ao longo de várias iterações
